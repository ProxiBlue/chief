## CRITICAL INSTRUCTIONS

You MUST follow this exact workflow:

1. Analyze the code diff for security vulnerabilities using the OWASP Top 10 (2021) framework:

   - **A01:2021 – Broken Access Control**: Missing or bypassed access controls, privilege escalation, CORS misconfiguration, path traversal, insecure direct object references (IDOR), missing function-level access control
   - **A02:2021 – Cryptographic Failures**: Weak or missing encryption, sensitive data transmitted in cleartext, hardcoded secrets/credentials, weak hashing algorithms (MD5/SHA1 for passwords), improper key management, missing TLS
   - **A03:2021 – Injection**: SQL injection, NoSQL injection, command/OS injection, XSS (reflected/stored/DOM), template injection, LDAP injection, header injection, log injection
   - **A04:2021 – Insecure Design**: Missing rate limiting, business logic flaws, missing abuse-case protections, trust boundary violations, insecure defaults
   - **A05:2021 – Security Misconfiguration**: Verbose error messages leaking internals, unnecessary features enabled, missing security headers, default credentials, overly permissive permissions, XML external entity (XXE) processing
   - **A06:2021 – Vulnerable and Outdated Components**: Known-vulnerable dependency patterns, unsafe use of external libraries, missing integrity checks on external resources
   - **A07:2021 – Identification and Authentication Failures**: Missing authentication checks, weak password policies, insecure session handling, credential stuffing exposure, missing MFA where expected
   - **A08:2021 – Software and Data Integrity Failures**: Insecure deserialization, prototype pollution, untrusted data in CI/CD pipelines, missing code/data integrity verification, auto-update without signature verification
   - **A09:2021 – Security Logging and Monitoring Failures**: Sensitive data in logs (PII, tokens, passwords), missing audit trails for security events, insufficient logging of auth failures
   - **A10:2021 – Server-Side Request Forgery (SSRF)**: Unvalidated URLs in server-side requests, missing allowlist for outbound requests, internal service exposure

   Additionally check for these cross-cutting concerns:
   - **Race conditions**: TOCTOU bugs, unsafe concurrent access to shared state
   - **Resource management**: Denial-of-service vectors, unbounded allocations, missing resource limits
   - **File system safety**: Path traversal, symlink attacks, insecure temporary files

## SCORING RULES

You MUST generate exactly one criterion for each OWASP category (and cross-cutting concern) that is relevant to the code diff. Use the OWASP ID in your criterion name for traceability. Skip categories that genuinely do not apply to the changes.

At minimum, always evaluate these if the diff touches any application logic:
- "A03:2021 No injection vulnerabilities" (SQL, command, XSS, etc.)
- "A02:2021 No sensitive data exposure" (secrets, credentials, PII in logs/errors)
- "A01:2021 Access controls are sufficient" (auth checks, authorization)

Score each criterion 1-10:
- 10 = no security concerns found for this category
- 7 = acceptable, minor observations but no exploitable issue
- 4 = potential vulnerability that should be reviewed
- 1 = critical security flaw, actively exploitable

For scores below 7, include a brief description with file path and line number explaining the vulnerability, its OWASP category, and potential impact.
